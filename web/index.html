快速开始

1. 获取 token

在 po0 控制台的机器详情页开启「防火墙加白」功能，复制生成的 API token（pgnfw_ 开头）。一台机器对应一个 token，多个 token 用英文逗号分割。

2. 选择你的客户端安装

S
SurgeiOS / Mac · sgmodule
一键安装

安装时在模块参数 tokens 填入 token。首页自带手动刷新面板。

L
LooniOS · plugin
一键安装

安装后进插件设置，在 API tokens 输入框填入 token。

St
StashiOS · stoverride
一键安装

安装后编辑覆写，把两处 argument: tokens= 补全。附手动刷新磁贴。

QX
Quantumult XiOS · task 片段
一键安装

QX 不支持脚本参数：用 BoxJs 给 key po0fw_tokens 写入 token，或本地脚本改 INLINE_TOKENS。

🚀
ShadowrocketiOS · 导入 Surge 模块

配置 → 模块 → 右上角 + → 粘贴链接安装，参数 tokens 填 token。

E
EgerniOS / Mac · 导入 Surge 模块

模块 → 添加 Surge 模块 → 粘贴链接，参数 tokens 填 token。

💡 一键按钮没反应？ 部分浏览器会拦截 App URL Scheme，点「复制链接」后到客户端里「从 URL 安装 / 导入」即可。

3. 完成，看面板

之后无需任何操作：切换网络时立即、以及每 3 分钟自动检查一次，出口 IP 变了才会真正写入白名单。支持面板的客户端会显示：

设计说明

  • 省坑位：每次先只读查询（GET），当前出口已在白名单就不写入（POST）。白名单上限 5 个，写满按写入时间先进先出淘汰；不必要的写入会把常用 IP 挤出去，所以先查后写很重要。
  • 蜂窝优化：蜂窝出口是 CGNAT，IP 变化频繁。自动加白在 24 小时内最多为蜂窝消耗 2 个坑位，超限后暂停并通知；手动刷新（面板/磁贴）不受限，需要连机器时点一下即可。
  • 安静运行：状态用持久化存储记账，只在失败、限频或消耗新坑位时才推送通知。
  • 隐私：面板不显示 token；token 只保存在你自己的客户端配置里，仓库与页面不包含、不上传任何 token。
⚠️ 注意：坑位满（5/5）后继续加白会按写入时间先进先出自动淘汰最旧的 IP，无需手动清理；但频繁写入会把还在用的 IP（比如家宽）挤出白名单 —— 蜂窝限频就是为了防这个。

FAQ

Q: 走代理时会把代理服务器 IP 加白吗？
不会。各载体都自带 console.po0.com 直连规则，API 请求始终直连，看到的是设备真实出口 IP。

Q: 家宽换 IP 后多久生效？
网络切换事件是即时的；纯粹的公网 IP 变更最迟 3 分钟内被 cron 捕捉。

Q: 不同客户端功能有差异吗？
核心加白逻辑完全一致。差异：不支持 $network 的客户端无法识别蜂窝（按非蜂窝处理）；QX 无面板与脚本参数（token 走存储 key）。

Q: 支持几台 po0 机器？
不限，tokens 逗号分割即可，面板按 #1 #2 … 分行显示。
